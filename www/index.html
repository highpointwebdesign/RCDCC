function Help: Tuning
keyboard_arrow_right
lock Servo Lock
Use the padlock in Servo Trim & Direction to freeze both the trim sliders and the CW or CCW direction buttons before a test drive.
Its purpose is to prevent accidental edits while scrolling, reviewing values, or handing the phone to someone else.
arrows_input Hardware Configuration
Select how your DCC is physically mounted. The arrow is printed on the box.
Correct orientation is critical for accurate roll and pitch readings, so change this any time the controller is re-mounted.
height Ride Height
Definition: Sets the suspension's resting position, controlling how high or low the chassis sits before the vehicle starts reacting to movement.
Formula: servoOffset = rideHeight × (maxServo - minServo) / 100
0% = Lowest ride height (maximum compression)
50% = Mid-range ride height (neutral)
100% = Highest ride height (maximum extension)
waves Damping
Definition: Controls how much the system resists vertical motion, smoothing bumps and limiting bounce after the chassis is disturbed.
Workflow: Start around the default and run over your roughest section. Increase damping if the truck keeps bouncing after bumps; reduce damping if it feels harsh and loses grip on choppy terrain.
Formula: verticalEffect = -verticalAccel × damping
0.1 = Soft damping (minimal vertical compensation)
0.8 = Default normal damping
2.0 = Stiff damping (maximum vertical compensation)
compress Stiffness
Definition: Sets how strongly the suspension counters body roll and pitch when the vehicle leans under cornering, braking, or acceleration.
Workflow: Tune stiffness after ride height and damping feel close. Raise it if body roll is excessive in turns; lower it if the chassis feels twitchy or over-corrects during quick transitions.
Formula: rollEffect = roll × stiffness, pitchEffect = pitch × stiffness
0.5 = Soft response (minimal roll/pitch correction)
1.0 = Default normal response
3.0 = Stiff response (maximum roll/pitch correction)
speed Reaction Speed
Definition: Determines how quickly suspension corrections are applied after motion is detected, balancing sharp response against smoothness.
Workflow: Set reaction speed to match terrain and driving style. Increase it for high-speed or aggressive driving where fast correction helps, and decrease it for crawling or mixed terrain to keep motion smoother.
Formula: smoothing = 1.0 / (1.0 + (5.0 / reactionSpeed))
0.1 = Very slow/smooth response (smoothing ≈ 0.02)
1.0 = Default balanced response (smoothing ≈ 0.17)
5.0 = Very fast/aggressive response (smoothing ≈ 0.5)
balance Balance (Rear/Front)
Definition: Splits pitch correction between the front and rear axles so you can favor nose control, rear stability, or an even mix.
Workflow: Use this after core response is dialed in. Shift toward front if the nose dives too much under braking, or toward rear if acceleration squat and rear instability are the bigger issue.
Formula: frontFactor = balance, rearFactor = 1.0 - balance
0% = All rear (front: 0%, rear: 100% pitch effect)
50% = Balanced (front: 50%, rear: 50% pitch effect)
100% = All front (front: 100%, rear: 0% pitch effect)
memory Sensor Refresh Rate
Controls sensor reading and suspension update frequency. Higher = faster response but more CPU load. 25 Hz is recommended.
5 Hz = Low/Conservative (less CPU load)
25 Hz = Recommended/Balanced (default)
50 Hz = High/Fast (maximum CPU load)

help Help: Lights
keyboard_arrow_down
  1. Set Total LED Count: Update the Total LEDs field in the Manage Light Groups card to match your LED count.
  2. Create Light Groups: Click "Add Light Group" to define a group with a name, pattern, and color(s).
  3. Assign LEDs to Groups: In the group editor, select which LED positions belong to this group.
  4. Set Brightness: Adjust the brightness slider for each group independently.
  5. Configure Patterns: Choose a pattern (solid, blink, pulse, etc.). Patterns marked with * use both primary and secondary colors.
  6. Organize Priority: Use up/down arrows in each group card to set priority order (top = highest priority).
  7. Master Switch: Available on the Dashboard to enable/disable all lights at once.
  8. Enable Individual Groups: Use the group toggles in the Manage Light Groups card to enable/disable specific groups.
  9. Save a Profile: Tap "+ Save as New" in the Lighting Profiles card to save your setup for quick recall.

help Help: Garage
keyboard_arrow_right
  1. Adding a vehicle — Tap the directions_car+ button in the card header to scan for a nearby ESP32 and add it to your garage.
  2. Connecting — Tap a vehicle card to connect. The card turns green when connected.
  3. Disconnecting — Long-press (hold) a connected card, or tap  ⋮  → Disconnect from the menu.
  4. Navigating — Tap  ⋮  → Suspension, Lights, or FPV to jump directly to that section for the selected vehicle.
  5. Renaming — Tap  ⋮  → Rename to give your vehicle a custom display name (up to 10 characters).
  6. About — Tap  ⋮  → About to see the device MAC ID and firmware details.
  7. Backup — Tap Backup to export your garage, light groups, and lighting profiles to a JSON file. Store it somewhere safe.
  8. Restore — Tap Restore to import a previously exported backup file. This replaces current data.
  9. Deleting a vehicle — Tap  ⋮  → Delete. All driving profiles, lighting profiles, and light groups saved for that truck are also removed.
