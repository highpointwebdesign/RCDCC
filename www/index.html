function Help: Tuning
keyboard_arrow_right
tune Mode
Definition: Selects how tuning values are driven.
Auto: Uses Drive Mode to automatically blend key tuning parameters for quick setup and easy profile switching.
Custom: Unlocks individual sliders so you can tune each parameter directly for maximum control.
Tip: Start in Auto to get close quickly, then switch to Custom for fine tuning once handling is stable.
directions_car Drive Mode
Definition: Master blends between fully passive (no servo movement) and fully re-active suspension response.
0% = Off — servos hold position, no re-active response
50% = Mixed — partial re-active response blended in
100% = Sport — full re-active suspension engaged
Tip: Start at 50% on a new build and increase once servo direction and range are confirmed correct.
arrows_outward Suspension Height
Definition: Sets the neutral resting position of all four servos — how high or low the chassis sits before any re-active correction is applied.
0% = Lowest (maximum compression)
50% = Mid-range neutral (default)
100% = Highest (maximum extension)
Tip: Set this with the vehicle on a flat surface. Adjust until the chassis sits at the desired ride height before tuning response.
Warning: If chassis is set at upper or lower limits, re-active correction will be limited.
balance Balance (Rear/Front)
Definition: Splits pitch correction between front and rear axles — favor nose control, rear stability, or an even mix.
Rear = All pitch correction at the rear axle
Balanced = Equal split front and rear (default)
Front = All pitch correction at the front axle
Tip: Dial this in last. Shift toward Front if the nose dives under braking; shift toward Rear if acceleration squat is the bigger issue.
settings_motion_mode Bounce Speed
Definition: The natural frequency of the suspension model — how quickly it wants to oscillate and recover after a disturbance.
Slow = Soft, floaty response (low frequency)
Balanced = Default recommended starting point
Fast = Snappy, stiff response (high frequency)
Tip: Increase for rock crawling where slow deliberate movement is needed; decrease for high-speed runs to keep corrections quick.
airwave Bounce Decay
Definition: Damping ratio — controls how quickly oscillations die out after the chassis is disturbed.
Bouncy = Underdamped, servos oscillate after bumps
Balanced = Critically damped, settles cleanly (default)
Firm = Overdamped, very little bounce but slower recovery
Tip: If the chassis keeps bobbing after a bump, increase decay. If it feels sluggish returning to neutral, decrease it.
arrows_outward Motion Amount
Definition: Scales the total servo travel used for active corrections — how dramatically the chassis moves in response to roll and pitch input.
Small = Subtle corrections, minimal visible movement
Normal = Default balanced travel (recommended)
Large = Exaggerated movement, maximum visual effect
Tip: Keep at Normal until other parameters feel right, then increase for more dramatic lean and squat effects.
noise_aware Noise Guard
Definition: Deadband threshold — sensor readings below this level are ignored, preventing servos from reacting to vibration, idle noise, or micro-movements.
Off = No filtering, reacts to all sensor input
Normal = Default recommended threshold
High = Only reacts to significant movement
Tip: If servos twitch or hunt while the vehicle is stationary, increase Noise Guard until they settle.
speed Reaction Speed
Definition: Low-pass filter speed — how quickly corrections ramp up after motion is detected. Balances sharp response against smooth movement.
Slow = Gradual, smooth corrections (heavy filtering)
Balanced = Default recommended starting point
Fast = Immediate, sharp corrections (minimal filtering)
Tip: Increase for high-speed bashing; decrease for crawling or mixed terrain to keep motion looking natural.
lock Noise Latch
Definition: Hysteresis band around the Noise Guard threshold — once the signal drops back inside the deadband, it must fall further before corrections stop. Prevents rapid on/off chatter at the edge.
Low = Corrections cut out as soon as signal re-enters deadband
Normal = Small buffer prevents rapid toggling (default)
High = Wide buffer, corrections linger slightly longer
Tip: Only adjust if servos are flickering rapidly at the edge of movement. Raise slightly until chatter stops.
arrows_input Help: Hardware Configuration
keyboard_arrow_right
info How It Works
  • Changes apply to the vehicle in real-time — adjust a setting and observe the effect immediately.
  • If something goes wrong, power cycle the RCDCC unit — it will revert to the last saved configuration automatically.
  • Once settings are dialed in, tap Save Settings. This becomes the new last known good and will be loaded automatically on the next power cycle.
  • The lock icon on each section prevents accidental changes.
  • Sensor Refresh Rate controls how often the accelerometer is sampled and suspension corrections are calculated. Higher rates respond faster but increase CPU load. 25 Hz (25 times per second) is the recommended default — only go higher if corrections feel sluggish, or lower to reduce heat/power consumption.
sync_alt Invert
Definition: Reverses the direction a servo moves in response to roll and pitch inputs.
Tip: Enable if a corner rises when it should drop. Use Servo Test (Raise/Lower) first to confirm which servos need inverting before a test drive.
tune Trim
Definition: Fine offset applied to a servo's neutral center position, independent of Suspension Height.
Tip: Use trim only for small mechanical offsets (e.g. a link rod that doesn't sit perfectly level). Large trim values are a sign that Min/Max need adjusting instead.
expand Min & Max
Definition: The microsecond pulse-width limits for each servo — the physical travel range the firmware is allowed to command.
Tip: Set Min and Max to the safe mechanical limits of each corner's linkage. Going beyond these risks binding gears or damaging components. Use Raise/Lower in Servo Test to find the limits safely.
info Help: Servo Test
keyboard_arrow_right
info Purpose
Use the Raise and Lower buttons to move each servo to its maximum or minimum travel limit. Limits are defined by the Min and Max values in Per-Servo Setup.
This confirms each servo is responding and moving in the correct direction. If a corner moves in the wrong direction, enable Invert for that servo in Hardware Configuration › Per-Servo Setup.
arrow_upward Raise
Commands the servo to its Max position. The corner should rise (extend upward).
Tip: If the corner drops instead of rises, enable Invert for that servo.
arrow_downward Lower
Commands the servo to its Min position. The corner should drop (compress downward).
Tip: If binding or resistance is felt before full travel, reduce Min or Max to stay within the safe mechanical range.

help Help: Lights
keyboard_arrow_down
  1. Enable Lighting: Turn on Lighting Enabled in Hardware Setup.
  2. Set LEDs Count: Match the LEDs Count slider to your strip length.
  3. Create Segments: In LED Mapping Step 1, tap + Segment and name each segment.
  4. Color Segments: In Step 1, tap a segment color circle to pick a color.
  5. Assign LEDs: In Step 2, tap LED positions to map them to the selected segment.
  6. Set Brightness and FX: Use Profile Look controls to tune brightness, effect, speed, and secondary color where supported.
  7. Save Mapping: Use Save in Hardware Setup after editing segment mapping.
  8. Master Switch: Use the top Lights toggle to quickly enable or disable lighting output.
  9. Save a Profile: Tap "+ Save as New" in the Lighting Profiles card to save your setup for quick recall.
